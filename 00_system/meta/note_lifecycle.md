# Note Lifecycle（笔记生命周期）

> 本文定义本 Obsidian Vault 中一条笔记从「产生」到「沉淀」再到「归档」的完整生命周期规则。
>
> 目标只有一个：**减少当下决策成本，避免长期混乱**。

---

## 一、设计原则（先看这个）

1. **任何时候都不要求“一步到位”**
2. **宁可晚整理，也不在当下纠结**
3. **结构稳定 > 内容完美**
4. **允许中间态长期存在**

一句话总结：

> 写的时候只管写，整理是另一个阶段的事情。

---

## 二、生命周期总览

```text
产生 → 临时存放 → 初步整理 → 结构化沉淀 → 可选归档
```

对应到 Vault 中的目录：

| 阶段    | 目录                     |
| ----- | ---------------------- |
| 产生    | 20_daily / 10_inbox    |
| 临时存放  | 10_inbox               |
| 初步整理  | 30_notes / 40_projects |
| 结构化沉淀 | 30_notes + 60_mocs     |
| 归档    | 90_archived            |

---

## 三、阶段 1：产生（Capture）

### 来源

笔记可能来自：

* Daily Note 中的随手记录
* 工作 / 学习过程中的想法
* 会议、讨论、灵感
* 阅读、视频、课程中的零散收获

### 规则

* **默认写在 Daily Note 中**
* 不要创建新文件
* 不要考虑分类
* 不要加复杂标签

Daily 的职责只有一个：

> 承接当天发生的一切

---

## 四、阶段 2：临时存放（Inbox）

### 何时进入 Inbox

当某一条内容满足以下任一条件：

* 不属于“今天”，但值得保留
* 需要以后再整理
* 不确定放在哪个目录
* 未来可能发展成一篇正式笔记

### 操作方式

* 从 Daily 中剪切 / 拆分
* 新建文件放入 `10_inbox`

### Inbox 的核心规则

> **Inbox 可以乱，但不能不动**

* 允许文件命名不完美
* 允许内容不完整
* 允许长期存在（但要定期回看）

禁止行为：

* 为了“找个好位置”而拖延记录

---

## 五、阶段 3：初步整理（Notes / Projects）

### 何时整理

推荐节奏：

* 每周 1 次（10–30 分钟）
* 或 Inbox 文件数量 > 10 条

### 整理决策顺序（非常重要）

只回答这一个问题：

> **这是“知识”还是“事情”？**

#### 1️⃣ 知识 → `30_notes`

* 技术原理
* 方法论
* 可复用经验
* 学习总结

#### 2️⃣ 事情 → `40_projects`

* 有明确目标
* 有上下文和期限
* 会持续推进

> 不要在 Notes 和 Projects 之间反复横跳，选一个就行。

---

## 六、阶段 4：结构化沉淀（Notes + MOCs）

### Notes 的要求

一条成熟的 Note 通常具备：

* 明确主题（一个问题 / 一个概念）
* 相对稳定（不会频繁改结构）
* 可被多处引用

但 **不强制一次完成**。

### MOC 的职责

`60_mocs` 中的文件：

* 不写细节内容
* 只做索引、导航、关系说明
* 回答：

> “这个领域，我都写过哪些东西？”

### 什么时候新建 MOC

* 某个目录下的 Note ≥ 5 篇
* 或你开始感觉“有点散，不好找”

---

## 七、阶段 5：归档（Archive）

### 什么应该被归档

* 已完成项目
* 明确不会再更新的内容
* 历史记录（但有保留价值）

### 归档规则

* 移动到 `90_archived`
* 不重命名
* 不重新整理结构

Archive 的目标：

> **只存，不打扰**

---

## 八、标签在生命周期中的角色

标签只用于：

* 状态（draft / idea / done）
* 属性（book / video / meeting）

禁止：

* 用标签代替目录
* 用标签做复杂分类

详细规则见：`meta/tags.md`

---

## 九、你只需要记住的 3 条铁律

1. **不确定 → Inbox**
2. **写作和整理分离**
3. **系统为你服务，不是你服务系统**

如果你哪天开始抗拒记笔记了，说明：

> 系统太复杂了，回到这三条。

---

## 十、版本说明

* 本规则允许演进
* 修改前先观察 2–4 周
* 禁止频繁“系统重构”

---

> 一个好的笔记系统，应该让你**越来越少思考“怎么记”**，而把注意力留给**真正重要的事情**。
